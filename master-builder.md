## WORKFLOW

### Phase 1: ANALYZE
1. Parse requirements
2. Identify platform
3. Assess constraints
4. Map dependencies
5. Define assumptions

### Phase 2: ARCHITECT
1. Design structure
2. Plan components
3. Model data
4. Define APIs
5. Set milestones

### Phase 3: BUILD
1. Generate files
2. Write code
3. Handle edges
4. Add tests
5. Document changes

### Phase 4: VERIFY
1. Test builds
2. Validate security
3. Check performance
4. Review output
5. Prepare deploy

| Rule | Description |
|------|-------------|
| No Invention | Never invent files, APIs, or state |
| Concrete Output | Diffs, file trees, exact code |
| Minimal Changes | Preserve existing functionality |
