## EXPLANATION STRATEGIES

### The ELI5 Approach (Explain Like I'm 5)
```
Rules:
• Use simple, everyday words
• Compare to things everyone knows
• Avoid abstract concepts
• Use short sentences
• Focus on "what" not "why it matters academically"

Template:
"Imagine [familiar thing]. [Concept] is kind of like
that, but [key difference]. So when [scenario],
it's because [simple mechanism]."
```

### The Feynman Technique
```
Steps:
1. Name the concept at the top of a page
2. Explain it in plain language as if teaching a child
3. Identify gaps in your explanation
4. Go back and simplify further
5. Use analogies to connect to known concepts

Result: If you can't explain it simply,
you don't understand it well enough.
```

### The Layer Method
```
LAYER 1: The Headline (1 sentence)
"[Concept] is basically [simplest definition]."

LAYER 2: The Paragraph (5 sentences)
Expand with the key "how" and "why"

LAYER 3: The Page (detailed explanation)
Add nuance, edge cases, and depth

LAYER 4: The Chapter (comprehensive)
Full technical detail for experts
```

### The Analogy Bridge
```
Structure:
"Think of [concept] like [familiar thing].

In [familiar thing], [familiar mechanism].

Similarly, in [concept], [parallel mechanism].

The main difference is [key distinction]."
```

## AUDIENCE CALIBRATION

### For Children (5-10)
```
Use:
• Concrete objects (toys, food, animals)
• Stories and characters
• "Magic" framing for abstract concepts
• Lots of "imagine if..."
• Questions to engage

Avoid:
• Numbers larger than 100
• Invisible/abstract forces
• Long explanations
• Technical terms
```

### For Teens (11-17)
```
Use:
• Pop culture references
• Technology comparisons
• "Real world" relevance
• Some technical terms (defined)
• Cause-and-effect reasoning

Avoid:
• Talking down
• Outdated references
• Too much background
• Pure abstraction
```

### For General Adults
```
Use:
• Everyday analogies (driving, cooking, shopping)
• Practical implications
• Common experience comparisons
• Building from intuition

Avoid:
• Unnecessary jargon
• Academic framing
• Assumed background knowledge
• Overly technical depth
```

### For Experts in Other Fields
```
Use:
• Cross-domain analogies
• Structural similarities
• Technical vocabulary (defined)
• Respect for their expertise

Avoid:
• Over-simplification
• Patronizing tone
• Hiding complexity
• Domain-specific jargon without definition
```

## SIMPLIFICATION TECHNIQUES

### Jargon Busting
```
When you encounter jargon:
1. Flag the term
2. Provide simple definition in parentheses
3. Use the simple version going forward

Example:
"Latency (the delay between request and response,
like how long it takes for a webpage to load)..."
```

### Chunking
```
Break complex processes into steps:
1. First, [initial action]
2. This causes [result]
3. Which then [next step]
4. Finally, [outcome]

Each step should be one simple idea.
```

### The "So What?" Test
```
After each point, ask "So what? Why does
this matter to the person?"

Add relevance:
"[Technical fact]. This means that [practical
implication for the audience]."
```

### Concrete Over Abstract
```
Abstract: "Exponential growth accelerates over time."

Concrete: "If you fold a piece of paper 42 times,
it would reach the moon. Each fold doubles the
thickness, so it grows incredibly fast."
```

## EXAMPLE EXPLANATIONS

### Blockchain
```
THE SHORT VERSION:
A shared notebook that everyone can read, but no
one can erase or change once something is written.

THE ANALOGY:
Imagine a classroom where every student has a copy
of the same notebook. When someone writes something
new, everyone copies it into their notebook at the
same time. If one student tries to change what they
wrote, everyone else's notebooks would disagree, so
the change gets rejected.

REAL EXAMPLE:
When you send Bitcoin, everyone's "notebook" records
that you sent it and someone else received it. No
bank needed - everyone keeps each other honest.
```

### Inflation
```
THE SHORT VERSION:
When money becomes worth less, so prices go up.

THE ANALOGY:
Imagine everyone in your town gets $1,000 extra.
Now everyone has more money to spend, so stores
raise prices because people can afford to pay more.
The money didn't get stronger - it got weaker.

REAL EXAMPLE:
If a coffee cost $2 last year and $2.20 this year,
that's 10% inflation. Your $100 now buys what $90
bought last year.
```
