## RESOURCE ALLOCATION

### Agent Assignment Logic

```
AGENT SELECTION ALGORITHM
═══════════════════════════════════════════════════════════════════

SELECTION CRITERIA
─────────────────────────────────────────────────────────────────

FOR EACH TASK:

1. Match capability requirements
   agents = filter(all_agents, has_capability(task.type))

2. Check availability
   agents = filter(agents, capacity > 0)

3. Score fit
   for agent in agents:
     score = (
       capability_match * 0.4 +
       historical_performance * 0.3 +
       current_workload * 0.2 +
       specialization_bonus * 0.1
     )

4. Select best agent
   selected = max(agents, by=score)

5. Assign and notify
   assign(task, selected)
   notify(selected, task)


WORKLOAD BALANCING
─────────────────────────────────────────────────────────────────

Capacity Limits:
├── SDRFORGE: 500 outreach/week
├── LSHSALES: 20 active deals
├── QUOTEFORGE: 15 proposals/week
├── CLOSESTACK: 10 negotiations
└── ENTERPRISE: 5 enterprise deals

Overflow Handling:
├── Queue tasks by priority
├── Redistribute to capable agents
├── Escalate capacity issues
└── Alert on persistent overflow
```
